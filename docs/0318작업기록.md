## 2. Git 환경 구성 완료

### 수행한 작업

1. `.gitignore` 생성 (Java/Maven/IDE/OS/env 전체 커버)
2. `git init` → `main` 브랜치 초기 커밋
3. `v0.0.0` 태그 (Claude 프로토타입 baseline)
4. `backup/claude-prototype` 브랜치 생성 (원본 보존)
5. `dev` 브랜치 생성 → 현재 작업 브랜치로 전환

### 최종 브랜치 구조

```
main                    ← 버전 릴리즈 전용 (v0.0.0 태그됨)
backup/claude-prototype ← 현재 상태 백업 (원본 보존)
* dev                   ← 개발 통합 브랜치 (현재 위치)
```

### 작업 흐름

```
dev
 ├── feat/xxx     ← 새 기능
 ├── refac/xxx    ← 리팩토링
 ├── fix/xxx      ← 버그 수정
 └── → dev 머지 → 검증 → main 머지 (v0.1.0, v0.2.0...)
```

## 3. 개발 로드맵 — 설계 목표까지의 단계

오늘 할 수 있는 현실적인 작업을 단계별로 정리하면:

| 순서 | 작업 | 내용 | 난이도 |
|---|---|---|---|
| **A** | 프로젝트 뼈대 재구성 | 패키지명 변경 (`com.myllm.financeai`), pom.xml 의존성 추가 (JPA, Security, Flyway, Lombok, Swagger), application.yml 전환 | 중 |
| **B** | DB + Flyway + 핵심 엔티티 | PostgreSQL 연결, Flyway V1~V3 (users, documents, document_chunks, price_daily, financial_metrics, analysis_sessions, analysis_results), BaseTimeEntity, 엔티티 클래스 | 대 |
| **C** | JWT 인증 | SecurityConfig, JwtTokenProvider, JwtAuthenticationFilter, signup/login API | 중 |
| **D** | 문서/가격/재무 CRUD API | DocumentController, MarketController + Repository + Service | 중 |
| **E** | 분석 세션 저장 API | AnalysisSession/Result 실제 DB 저장 + 세션 목록/상세 조회 | 중 |

A → B → C가 토대, D → E가 그 위에 올라가는 기능.
